# OpenTelemetry Collector Contrib Go 代码规范

## 一、代码风格

### 格式化
- 强制使用 `gofmt`，提交前执行 `make golint` 或子模块 `make lint`。
- 单行最长 120 字符，超长时按逻辑断行。
- 所有注释句末加句号。
- 导出类型/函数必须有 godoc 风格注释（首行以被注释名开头）。

### Happy Path（直线流）
主流程代码直线向下，错误立即处理，避免主逻辑嵌套在 `if err == nil` 内：

```go
// 正确：错误前置，主流程直线
func (e *fileExporter) consumeTraces(_ context.Context, td ptrace.Traces) error {
    buf, err := e.marshaller.marshalTraces(td)
    if err != nil {
        return err
    }
    return e.writer.export(buf)
}

// 错误：主逻辑嵌套
func (e *fileExporter) consumeTraces(_ context.Context, td ptrace.Traces) error {
    buf, err := e.marshaller.marshalTraces(td)
    if err == nil {
        return e.writer.export(buf)
    }
    return err
}
```

### 错误处理
- 所有错误用 `fmt.Errorf("操作描述: %w", err)` 包装并带上下文。
- 包级别定义哨兵错误，用 `errors.Is` / `errors.As` 判断。
- 不允许裸 `return err`（除非已有充足上下文）。
- 禁止 `panic`，除非是程序初始化阶段不可恢复的错误。

```go
// 哨兵错误定义（config.go 或 errors.go）
var (
    errInvalidPath       = errors.New("path must be non-empty")
    errAppendCompression = errors.New("append and compression cannot be enabled at the same time")
)

// 错误包装
func newMarshaller(conf *Config, host component.Host) (*marshaller, error) {
    if conf.Encoding != nil {
        enc := host.GetExtensions()[*conf.Encoding]
        if enc == nil {
            return nil, fmt.Errorf("unknown encoding %q", conf.Encoding)
        }
    }
    return &marshaller{}, nil
}

// errors.Is / errors.As 使用
if errors.Is(err, context.Canceled) {
    return fmt.Errorf("operation cancelled: %w", err)
}
```

## 二、命名规范

### 变量 / 参数
- 使用有意义的英文词，避免 Go 预定义标识符（`len`, `new`, `make`, `error`, `string` 等）。
- 短变量（循环索引等）可用 `i`, `k`, `v`；临时 context 用 `ctx`。
- 无需显式命名的参数用 `_` 占位：`func (r *receiver) Shutdown(_ context.Context) error`。

```go
// 正确
func process(name string, size int, shouldCreate bool) error { ... }

// 错误：shadowing 内建标识符
func process(new string, len int, make bool) error { ... }
```

### 函数 / 方法
- 工厂函数统一命名 `NewFactory() xxx.Factory`。
- 创建子函数：`createDefaultConfig()`、`createTracesExporter()`、`createLogsReceiver()` 等。
- 组件方法：`Start(ctx, host)`、`Shutdown(ctx)`、`Capabilities()`。
- 配置验证：`(cfg *Config) Validate() error`。

### 结构体 / 类型
- 名词，驼峰，首字母大写导出；私有类型首字母小写。
- 组件实现结构通常与包同名（如 `fileExporter`、`fluentReceiver`）。

### 常量 / 错误
- 哨兵错误：`errXxx`（包级私有）或 `ErrXxx`（需导出时）。
- 常量：`formatTypeJSON`、`defaultMaxBackups` 等小驼峰私有，或 `ALL_CAPS` 用于跨包枚举。

### 包名
- 全小写、无下划线，体现功能（`fileexporter`、`attributesprocessor`）。
- 子包 `internal/metadata` 存放 `mdatagen` 生成的文件，不手动修改。

## 三、组件结构规范

每个组件目录的标准文件布局：

```
receiver/fooreceiver/
  ├── metadata.yaml               # 组件元数据（类型、稳定性、codeowner）
  ├── factory.go                  # NewFactory() + create*函数
  ├── config.go                   # Config 结构体 + Validate()
  ├── receiver.go                 # 核心实现
  ├── README.md                   # 使用说明和配置示例
  ├── testdata/
  │   └── config.yaml             # 测试用配置
  ├── *_test.go                   # 测试文件
  └── internal/metadata/
      └── generated_status.go     # 由 mdatagen 生成，勿手动修改
```

### metadata.yaml（组件元数据）

```yaml
type: file
display_name: File Exporter

status:
  class: exporter
  stability:
    alpha: [traces, metrics, logs]
    development: [profiles]
  distributions: [core, contrib]
  codeowners:
    active: [githubusername]
```

生成的 `internal/metadata/generated_status.go`：

```go
// Code generated by mdatagen. DO NOT EDIT.
package metadata

import "go.opentelemetry.io/collector/component"

var (
    Type      = component.MustNewType("file")
    ScopeName = "github.com/open-telemetry/opentelemetry-collector-contrib/exporter/fileexporter"
)

const (
    TracesStability   = component.StabilityLevelAlpha
    MetricsStability  = component.StabilityLevelAlpha
    LogsStability     = component.StabilityLevelAlpha
    ProfilesStability = component.StabilityLevelDevelopment
)
```

### factory.go 标准结构

```go
func NewFactory() exporter.Factory {
    return xexporter.NewFactory(
        metadata.Type,
        createDefaultConfig,
        xexporter.WithTraces(createTracesExporter, metadata.TracesStability),
        xexporter.WithMetrics(createMetricsExporter, metadata.MetricsStability),
        xexporter.WithLogs(createLogsExporter, metadata.LogsStability),
    )
}

func createDefaultConfig() component.Config {
    return &Config{
        FormatType: formatTypeJSON,
        Rotation:   &Rotation{MaxBackups: defaultMaxBackups},
    }
}

// 创建函数签名统一
func createTracesExporter(
    ctx context.Context,
    set exporter.Settings,
    cfg component.Config,
) (exporter.Traces, error) {
    oCfg := cfg.(*Config)
    exp := newExporter(oCfg, set.Logger)
    return exporterhelper.NewTraces(ctx, set, cfg, exp.consumeTraces,
        exporterhelper.WithStart(exp.Start),
        exporterhelper.WithShutdown(exp.Shutdown),
        exporterhelper.WithCapabilities(consumer.Capabilities{MutatesData: false}),
    )
}
```

### config.go 标准结构

```go
// Config 定义组件的配置。
type Config struct {
    Path        string        `mapstructure:"path"`
    FormatType  string        `mapstructure:"format"`
    Compression string        `mapstructure:"compression"`
    FlushInterval time.Duration `mapstructure:"flush_interval"`
}

// 编译期确保实现 component.Config 接口
var _ component.Config = (*Config)(nil)

// Validate 校验配置合法性。
func (cfg *Config) Validate() error {
    if cfg.Path == "" {
        return errInvalidPath
    }
    if cfg.FormatType != formatTypeJSON && cfg.FormatType != formatTypeProto {
        return errors.New("format type is not supported")
    }
    if cfg.FlushInterval < 0 {
        return errors.New("flush_interval must be non-negative")
    }
    return nil
}
```

## 四、日志规范（zap）

Logger 从 `*.Settings` 中获取，禁止直接构造 `zap.Logger`：

```go
type fooReceiver struct {
    logger *zap.Logger
    // ...
}

func newFooReceiver(set receiver.Settings, cfg *Config) (*fooReceiver, error) {
    return &fooReceiver{
        logger: set.Logger,
    }, nil
}

// 使用结构化字段，不拼字符串
func (r *fooReceiver) Start(ctx context.Context, _ component.Host) error {
    r.logger.Info("Starting receiver.", zap.String("endpoint", r.cfg.Endpoint))
    if err := r.listen(); err != nil {
        r.logger.Error("Failed to start listener.", zap.Error(err))
        return fmt.Errorf("start listener: %w", err)
    }
    return nil
}
```

日志级别选择：
- `Debug`：调试信息，生产环境默认关闭。
- `Info`：关键生命周期事件（Start/Shutdown）。
- `Warn`：可恢复的异常情况。
- `Error`：不可恢复的错误（通常同时返回 `error`）。

## 五、pdata（Pipeline Data）操作规范

信号类型从对应子包导入：

```go
import (
    "go.opentelemetry.io/collector/pdata/plog"
    "go.opentelemetry.io/collector/pdata/pmetric"
    "go.opentelemetry.io/collector/pdata/ptrace"
    "go.opentelemetry.io/collector/pdata/pcommon"
)
```

Metrics 层级遍历示例：

```go
func processMetrics(md pmetric.Metrics) {
    for i := 0; i < md.ResourceMetrics().Len(); i++ {
        rm := md.ResourceMetrics().At(i)
        attrs := rm.Resource().Attributes()  // pcommon.Map
        for j := 0; j < rm.ScopeMetrics().Len(); j++ {
            sm := rm.ScopeMetrics().At(j)
            for k := 0; k < sm.Metrics().Len(); k++ {
                m := sm.Metrics().At(k)
                switch m.Type() {
                case pmetric.MetricTypeGauge:
                    m.Gauge().DataPoints()
                case pmetric.MetricTypeSum:
                    m.Sum().DataPoints()
                case pmetric.MetricTypeHistogram:
                    m.Histogram().DataPoints()
                }
            }
        }
    }
}
```

序列化：

```go
// JSON / Proto marshaler 通过 map 选择
var traceMarshalers = map[string]ptrace.Marshaler{
    "json":  &ptrace.JSONMarshaler{},
    "proto": &ptrace.ProtoMarshaler{},
}
```

## 六、Feature Gate 规范

在独立文件（如 `featuregates.go` 或 `internal/featuregates/`）中定义，勿散落在业务代码中：

```go
import "go.opentelemetry.io/collector/featuregate"

// 命名格式：<component>.<FeatureName>FeatureGate
var UseNewParserFeatureGate = featuregate.GlobalRegistry().MustRegister(
    "receiver.fooreceiver.useNewParser",     // 门 ID：<class>.<component>.<feature>
    featuregate.StageAlpha,                  // Alpha / Beta / Stable
    featuregate.WithRegisterDescription("When enabled, uses the new parser implementation."),
    featuregate.WithRegisterFromVersion("v0.100.0"),
)

// 使用
if UseNewParserFeatureGate.IsEnabled() {
    // 新路径
}
```

阶段定义：
- `StageAlpha`：默认关闭，可手动开启。
- `StageBeta`：默认开启，可手动关闭。
- `StageStable`：始终开启，即将移除开关。

## 七、测试规范

### 文件命名
- `config_test.go`、`factory_test.go`、`<component>_test.go`，与被测文件同目录。
- 测试辅助数据放在 `testdata/` 子目录。

### 测试函数命名
- `Test<功能名>`，如 `TestLoadConfig`、`TestCreateTracesExporter`、`TestValidate`。
- Benchmark：`Benchmark<功能名>`。
- 可并行测试加 `t.Parallel()`。

### Table-Driven 测试（优先使用）

```go
func TestValidate(t *testing.T) {
    t.Parallel()
    tests := []struct {
        name    string
        cfg     *Config
        wantErr string
    }{
        {
            name:    "valid config",
            cfg:     &Config{Path: "/tmp/out.json", FormatType: formatTypeJSON},
        },
        {
            name:    "missing path",
            cfg:     &Config{FormatType: formatTypeJSON},
            wantErr: "path must be non-empty",
        },
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            t.Parallel()
            err := tt.cfg.Validate()
            if tt.wantErr != "" {
                require.EqualError(t, err, tt.wantErr)
            } else {
                require.NoError(t, err)
            }
        })
    }
}
```

### Factory 测试模板

```go
func TestCreateDefaultConfig(t *testing.T) {
    cfg := createDefaultConfig()
    assert.NotNil(t, cfg)
    assert.NoError(t, componenttest.CheckConfigStruct(cfg))
}

func TestCreateTracesExporter(t *testing.T) {
    cfg := &Config{Path: t.TempDir() + "/out.json", FormatType: formatTypeJSON}
    exp, err := createTracesExporter(t.Context(),
        exportertest.NewNopSettings(metadata.Type), cfg)
    require.NoError(t, err)
    require.NoError(t, exp.Shutdown(t.Context()))
}
```

### 测试库
- `require.*`：断言失败立即终止测试。
- `assert.*`：断言失败继续执行以收集更多信息。
- `componenttest`、`exportertest`、`receivertest`：官方 nop 组件测试工具。

## 八、设计理念

| 原则 | 说明 |
|------|------|
| 简洁优先 | 避免过度设计，不为假设需求增加复杂性 |
| 零值可用 | 结构体零值应能直接使用，减少强制初始化 |
| 接口最小化 | 接口只暴露最小必要方法（Go 谚语："接口越大，抽象越弱"） |
| 组合优于继承 | 通过接口嵌入和结构体组合复用行为 |
| 并发安全 | 优先 channel 传递数据，必须共享时用 mutex，明确注释 goroutine 生命周期 |
| 可观测性 | 组件必须接入结构化日志（zap）、指标（otel metrics）和链路追踪 |
| 稳定性分级 | 每个信号（Traces/Metrics/Logs）独立声明稳定性：Development → Alpha → Beta → Stable |
| 生成代码不修改 | `internal/metadata/generated_*.go` 由 `mdatagen` 生成，必须通过 `metadata.yaml` 修改 |
| 测试覆盖强制 | 所有 PR 必须包含验证性测试，不接受无测试的功能代码 |
